# Stakeholder Analysis

| Stakeholder | Role | Key Concerns | Pain Points | Success Metrics |
|------------|------|-------------|------------|----------------|
| Students with Disabilities | Primary users who access learning materials | Accessibility, ease of use, personalized learning | Platforms not accessible, lack of support tools | Improved access, increased course completion rates |
| Lecturers | Provide content and monitor student progress | Student engagement, tracking performance | Difficult to identify struggling students | Improved student performance and engagement |
| University Administrators | Oversee academic systems and policies | Inclusive education compliance, system effectiveness | Lack of data on student performance | Improved graduation rates and inclusivity |
| IT Staff | Maintain and deploy the system | System reliability, security, scalability | System downtime, integration issues | 99% uptime, smooth system performance |
| Disability Support Services | Support students with special needs | Accessibility tools, assistive technologies | Limited adaptive tools in current systems | Increased student satisfaction and support usage |
| System Developers | Design and improve the system | Maintainability, scalability, performance | Complex system updates | Efficient updates and fewer system errors |
