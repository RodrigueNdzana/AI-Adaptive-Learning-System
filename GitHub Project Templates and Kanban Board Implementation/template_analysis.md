# Template Analysis and Selection

## Comparison of GitHub Project Templates

| Template | Columns / Workflow | Automation Features | Agile Suitability |
|----------|------------------|---------------------|------------------|
| Kanban | To Do → In Progress → Done | Manual or optional automation | Good for continuous workflow and task tracking |
| Team Planning | Backlog → Ready → In Progress → Done | Supports iteration planning and prioritization | Ideal for Scrum-based Agile teams |
| Feature Release | Planned → In Development → Testing → Released | Tracks feature lifecycle with milestone integration | Suitable for release management in Agile |
| Bug Tracker | New → Triaged → In Progress → Fixed → Closed | Label-based filtering and issue tracking | Best for bug management, not full Agile workflow |
| Iterative Development | Backlog → Iteration → In Progress → in Review → Done | Supports sprint cycles and iteration tracking | Highly suitable for Agile and incremental delivery |
| Product Launch | Planning → Development → Marketing → Launch → Post-Launch | Tracks cross-team coordination | Less suitable for development-focused Agile workflows |
| Roadmap | Ideas → Planned → In Progress → Completed | Long-term planning visualization | Useful for strategic planning, not daily task tracking |
| Team Retrospective | Went Well → Needs Improvement → Action Items | Supports feedback collection | Supports Agile reflection, not task execution |

---

## Selected Template: Iterative Development

### Justification

The **Iterative Development template** was selected because:

- It aligns strongly with **Agile principles**, especially incremental development.
- It supports **sprint-based workflows**, which matches Assignment 6.
- It allows continuous improvement through iterations.
- It integrates well with **user stories and backlog management**.
- It supports tracking of progress from development to review and completion.

Compared to other templates:

- **Kanban** is simpler but lacks structured iteration tracking.
- **Team Planning** is useful but less flexible for continuous iteration.
- **Bug Tracker** focuses only on issues, not full development.
- **Roadmap** and **Product Launch** are more strategic than operational.

Therefore, Iterative Development is the best fit for managing the AI Adaptive Learning System using Agile methodology.
